¿Qué es Git?

Git es un sistema de control de versiones distribuido, diseñado para manejar proyectos de software de manera eficiente y con alta velocidad. Creado por Linus Torvalds en 2005, permite a los desarrolladores seguir y gestionar los cambios en el código fuente a lo largo del tiempo.

Git y GitHub: una sinergia para el desarrollo colaborativo

GitHub es una plataforma de desarrollo colaborativo basada en Git. Mientras que Git es una herramienta de control de versiones distribuido, GitHub proporciona un entorno para alojar repositorios de Git en la nube, facilitando la colaboración y el seguimiento de proyectos.

Comandos Git

Configuración

  • config - obtiene y establece las opciones globales o de repositorio de git.

Creación y clonación de proyectos

  • init - crea un repositorio vacío o inicializa un repositorio existente.
  • clone - clona un repositorio en un nuevo repositorio.

Control de cambios

  • status - muestra el estado actual del repositorio.

Ramas y Fusiones

  • log - muestra un registro con los commits realizados en la rama actual en orden cronológico inverso.
    • [--oneline]
      Muestra la primera línea de cada commit omitiendo autor, fecha y descripción.
    • [--graph --all]
      Muestra una visualización en árbol de los commits en todas las ramas o de los commits en la rama actual si se excluye [--all].
